# Build & Tooling Configuration (`app/`)

## Responsibility

Configure and orchestrate the build pipeline, dev server, PWA service worker, TypeScript compilation, and code quality tooling for the OPTCG Lib React application. This layer owns the build-time environment, not the runtime application logic.

## Design Patterns

### Vite-based Build
- **Vite 8** with `@vitejs/plugin-react` (uses Oxc compiler for fast transforms)
- **Tailwind CSS v4** integrated via `@tailwindcss/vite` plugin (no PostCSS config needed)
- **`define` block** injects `__BUILD_TIME__` as the build timestamp (ISO string) into the application bundle

### PWA with Workbox
- **`vite-plugin-pwa`** configured with `registerType: 'autoUpdate'` — service worker auto-installs on update
- **Precaching** via `workbox.globPatterns: ['**/*.{js,css,html,ico,png,svg,webp,wasm,db}']`
- **12MB limit** on cached assets (`maximumFileSizeToCacheInBytes`) to accommodate the `sql.js` WASM (~1.5MB) and `optcg.db` (~10.6MB)
- **Runtime caching** for card images from `serveproxy.com` using `CacheFirst` strategy:
  - 5000-entry LRU cache, 180-day TTL
  - Dedicated `card-images` cache name
- **`clientsClaim: true` and `skipWaiting: true`** — service worker takes control immediately
- **`cleanupOutdatedCaches: true`** — removes old precaches on version change
- **PWA manifest** configured with standalone display, Android/iOS icons, and an OG-sized screenshot

### Strict TypeScript
- **TypeScript 6.0** with project references (`tsconfig.app.json` + `tsconfig.node.json`)
- **`tsconfig.app.json`**: app source (`src/`) — target `es2023`, `jsx: "react-jsx"`, bundler module resolution, strict dead-code checks (`noUnusedLocals`, `noUnusedParameters`)
- **`tsconfig.node.json`**: Vite config (`vite.config.ts`) — target `es2023`, Node types
- **`verbatimModuleSyntax: true`** — type imports must use `import type`
- **`noEmit: true`** — TypeScript is transpilation-only, Vite/esbuild handles output

### ESLint
- **Flat config** (`eslint.config.js`) with `@eslint/js` recommended base
- **`typescript-eslint` recommended** rules applied to `**/*.{ts,tsx}`
- **`eslint-plugin-react-hooks`** and **`eslint-plugin-react-refresh`** for React-specific linting
- **`globalIgnores(['dist'])`** — no linting of build output
- **`globals.browser`** for browser API recognition
- **`lint-staged`** configured to run `eslint --fix` on staged `src/**/*.{ts,tsx}` files
- **Husky** git hooks installed via `prepare` script (`../husky app/.husky`)

## Data & Control Flow

### Build Pipeline
```
Source files (src/)
  │
  ├── TypeScript (tsc -b)           ──→ type-check both tsconfig references
  │                                       (no emit; errors fail build)
  │
  ├── Vite/Rollup bundle            ──→
  │   ├── @vitejs/plugin-react      ──→ JSX transform via Oxc
  │   ├── @tailwindcss/vite         ──→ Tailwind CSS processing
  │   └── VitePWA (workbox)         ──→ precache manifest + SW generation
  │                                       runtime caching rules injected
  │
  └── Output (dist/)
      ├── index.html (injected hashed assets)
      ├── assets/*.{js,css}
      ├── optcg.db                   ──→ copied from public/ (precached)
      ├── sql-wasm-browser.wasm       ──→ from public/ (precached)
      ├── favicon.ico, *.png, icons  ──→ from public/ (precached)
      ├── og-image.png               ──→ from public/ (precached)
      └── sw.js (service worker)      ──→ generated by workbox
```

### Dev Server
- `vite dev` — starts Vite dev server with HMR
- Tailwind CSS and React transforms applied on-the-fly
- `optimizeDeps.include: ['sql.js']` — pre-bundles the sql.js ESM-compatible wrapper for fast cold starts

### Production Build
- `tsc -b && vite build` — type-check first, then bundle
- `vite preview` — serve production build locally for testing
- Workbox generates `sw.js` and precache manifest at build time

## Integration Points

### Serves the React App
- **`index.html`** — Entry point: mounts `<div id="root">`, loads `src/main.tsx` as module
- Preconnects to Google Fonts for *Barlow Semi Condensed* (600/700/800 weights)
- Open Graph and Twitter Card meta tags for social sharing
- Cloudflare Web Analytics beacon loaded from `static.cloudflareinsights.com`
- Theme color set to `#0f1117` (dark background)

### Manages the Service Worker
- **`VitePWA` plugin** generates and injects the service worker into the build
- **Precaching** ensures the app shell, WASM binary, and SQLite DB are available offline
- **Runtime caching** (`workbox.runtimeCaching`) enables offline-friendly card image loading from the image proxy
- **`autoUpdate` registration** — users always get the latest version without manual refresh
- **Pre-bundled `sql.js`** ensures the WASM SQLite engine loads efficiently in modern browsers

### Entry Expectations
- `app/public/optcg.db` must exist (copied from root after seeding)
- `app/public/sql-wasm-browser.wasm` must exist (auto-copied by build process)
- `vendor/punk-records/` is the data source (git submodule/clone, not tracked in Vite repo)
- Static assets (favicons, og-image, icons) must be present in `public/`
